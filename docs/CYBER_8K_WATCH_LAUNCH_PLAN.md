# Cyber 8-K Watch Launch Plan

This plan keeps the next phase product-focused. The goal is not another refactor; it is to learn whether evidence-backed cyber disclosure data saves real users time.

## MVP Scope

- 50-100 reviewed cyber disclosures
- event pages
- evidence-backed fields
- amendment tracking
- peer comparison
- CSV/API export
- weekly digest

## Target Users

- securities and corporate lawyers
- cyber incident response firms
- cyber insurers and risk teams
- investor relations and communications advisors
- journalists and researchers

## Interview Script

- How do you track cyber disclosures today?
- What fields do you manually extract?
- How do you compare one disclosure against peers?
- What evidence would make you trust an automated extraction?
- Would alerts, export, or API access save you time?

## Task-Based User Tests

- Find disclosures mentioning third-party vendors.
- Find filings later amended.
- Compare this filing to peers.
- Export a table for a memo.

## Pass / Fail Criteria

- 20 interviews
- 5 requests for alerts, export, or API access
- 2 serious pilot requests

## Pricing Hypotheses

- free public index
- paid reviewed dataset/export
- paid alerts/API
- custom benchmark reports

## Kill Criteria

- users like it intellectually but do not ask for data, alerts, or export
- review effort is too high per useful event
- existing tools already satisfy the workflow

## Later Verticals

Only add new verticals after Cyber 8-K Watch validates:

- accounting restatements
- auditor changes
- going concern
- regulatory enforcement
- product recalls
